Описание архитектуры:
Приложение спроектировано как легковесная служба мониторинга на базе шаблона .NET Worker Service.

Взаимодействие компонентов построено по следующей схеме:[Windows Service / Console]
           |
           |
      [Program] (Инициализация хоста, DI-контейнера и Serilog)
           |
           |
       [Worker] (Управление жизненным циклом и асинхронным циклом опроса)
           |
           |-->[SystemInfoCollector] (Сбор системных параметров ОС, дисков и процессов через API .NET)
           |
           |--> [ApiSender] (Асинхронная отправка JSON через HttpClient с контролем таймаутов)

Program.cs — настраивает конфигурацию (appsettings.json), инициализирует логирование (Serilog) и регистрирует приложение в системе служб Windows через .UseWindowsService(). 
Worker.cs — фоновый процесс, управляющий бесконечным циклом опроса системы с заданным интервалом.
SystemInfoCollector.cs — асинхронно собирает параметры железа и ОС. 
ApiSender.cs — сериализует модель SystemParameters в формат camelCase и выполняет HTTP POST-запросы.
AppSettings.cs и SystemParameters.cs  — классы для маппинга конфигурации JSON и формирования структуры отправляемого пакета системных параметров.
